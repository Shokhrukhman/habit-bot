# Habit Tracker Telegram Bot

Телеграм-бот для трекинга привычек с app-like интерфейсом: пользователь работает в одном "главном" сообщении, а бот перерисовывает экраны через inline-кнопки.

Проект ориентирован на практичный MVP:
- быстрое добавление привычек;
- одно ежедневное время напоминания на привычку;
- отметки `Done / Skip`, snooze и итог дня;
- удобная аналитика по дням и диапазонам.

## Что умеет бот

### 1) Привычки

- Раздел `📋 Habits` открывает сразу список привычек (без лишнего промежуточного экрана).
- Можно:
  - добавить привычку;
  - открыть карточку привычки;
  - активировать/деактивировать привычку;
  - задать/изменить время напоминания (`HH:MM`).
- Для каждой привычки хранится ровно одно время напоминания.

### 2) Напоминания

- Бот отправляет ежедневные напоминания в локальном часовом поясе пользователя.
- Для напоминания доступны действия:
  - `✅ Done`
  - `⏭ Skip`
  - `📝 Snooze`
- Snooze настраивается в минутах:
  - пресеты: 10, 15, 30, 60, 120;
  - произвольное значение: от 1 до 240.

### 3) Аналитика

- Раздел `📊 Stats` по умолчанию показывает последние 7 дней.
- Доступен выбор произвольного периода через мини-календарь (до 31 дня).
- Доступен экран `Day details` по выбранной дате.
- В прогрессе учитывается только статус `done`:
  - `skip` трактуется как `not done`.

### 4) Итоги дня

- В конце дня бот формирует и отправляет сводку по привычкам пользователя.
- Логи сохраняются по дате в локальной таймзоне.

## UX и навигация

- Используется один основной screen message на пользователя.
- Переходы между экранами идут через редактирование этого сообщения.
- Есть история экранов (`Back`) и быстрый переход `Home`.
- Если `screen_message_id` устарел (например, чат был очищен), бот автоматически создает новое главное сообщение и продолжает работу.
- Команды пользователя в личном чате удаляются для чистоты истории.

## Технологический стек

- Python 3.12
- aiogram v3 (async)
- PostgreSQL
- SQLAlchemy 2.0 (async)
- Alembic
- APScheduler (AsyncIOScheduler)
- `zoneinfo` для IANA-таймзон
- конфигурация через `.env` + `python-dotenv`

## Данные и ограничения

- `users`:
  - `telegram_id`
  - `timezone`
  - `snooze_minutes`
- `habits`:
  - название
  - активность
- `habit_reminder_times`:
  - одно время на привычку (UNIQUE по `habit_id`)
- `habit_logs`:
  - статусы по датам (`done`, `skip`, `not_done`)
- `ui_state`:
  - текущий экран
  - стек навигации
  - payload экрана
  - `screen_message_id`

## Быстрый старт

### 1) Установка зависимостей

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2) Подготовка `.env`

```bash
cp .env.example .env
```

Заполните:
- `BOT_TOKEN`
- `DATABASE_URL`

### 3) Запуск PostgreSQL

```bash
docker compose up -d
```

### 4) Применение миграций

```bash
alembic upgrade head
```

### 5) Запуск бота

```bash
python -m src.bot
```

## Проверка окружения

Быстрая проверка импортов и подключения к БД без запуска polling:

```bash
python -m src.bot --check
```
